# ToraSEO App 0.0.8 - единый релиз, надежный Codex bridge и полировка native chat

App 0.0.8 продолжает двухрежимную модель из `0.0.7`, но делает релизную упаковку и рабочие сценарии заметно спокойнее. Теперь публичная точка входа одна: релиз приложения, к которому прикрепляются installer assets, Claude Bridge Instructions ZIP и Codex Workflow Instructions ZIP.

## Главное

- Объединена публичная release entry для desktop app, Claude Bridge Instructions ZIP и Codex Workflow Instructions ZIP
- Исправлена доставка результатов Codex bridge: приложение наполняет `Overview` и `Confirmed facts` из структурированных данных инструментов, а не из текста в чате
- Завершенные bridge-результаты остаются видимыми после очистки временного scan state
- Улучшен UX Codex setup: постоянная подсказка после копирования prompt, feedback при копировании и более понятная логика approval
- Улучшены native AI chat ответы, учет выбранного языка и автоматическая интерпретация после scan
- Добавлена проверка каждой модели отдельно: приложение отличает реальное подключение модели от поддержки structured audit output
- Усилена безопасность MCP mismatch: protocol token больше не показывается в user-visible ответах

## Что входит в релиз

- Desktop installer assets
- `toraseo-claude-bridge-instructions-*.zip`
- `toraseo-codex-workflow-*.zip`

Начиная с `0.0.8`, эти assets должны публиковаться вместе в одном app release. Старые отдельные releases для instruction packages остаются историческими downloads, но больше не являются основной публичной точкой установки.

## Установка и обновление

- Новым пользователям стоит начинать с desktop app asset на странице релиза.
- Для сценария `MCP + Instructions` по-прежнему нужны MCP server и соответствующий instructions ZIP в AI client.
- Для сценария `API + AI Chat` нужно настроить provider и модель в Settings приложения.
- В Codex bridge лучше выбирать one-time chat/session MCP approval, если Codex client показывает такую опцию.
- При обновлении с `0.0.7` стоит заменить app build и скачать свежие instruction ZIPs из этого же release.

## Что изменилось

### App

- Bridge-mode результаты теперь мапятся из `buffer[toolId].data` в `Overview` и `Confirmed facts`.
- Severity summaries понимают и новый контракт `issues[]`, и старое имя `verdicts[]`.
- Completed bridge results сохраняются в UI после очистки временного state file.
- `Copy setup prompt` показывает постоянную Codex-подсказку рядом с sidebar, пока пользователь не закроет ее или пока реальные bridge data не придут в приложение.
- Копирование prompt теперь дает короткий feedback.
- Native AI chat получил более строгую locale-инструкцию, более полный формат ответа и автоматическую post-scan интерпретацию.
- OpenRouter settings теперь проверяют каждую сохраненную модель отдельно и показывают, вернула ли модель structured audit report или только обычный текстовый ответ.
- OpenRouter diagnostics теперь показывают usage/cost от провайдера, когда OpenRouter возвращает эти данные.

### MCP / bridge

- Codex workflow guidance явно направляет пользователя к one-time chat/session approvals, когда клиент это поддерживает.
- Bridge mismatch больше не раскрывает ожидаемые или полученные protocol token values в MCP responses и app-visible error text.
- Renderer-visible prompt text по-прежнему не содержит protocol token values.

### Docs / packaging

- Публичная упаковка релиза теперь app-led: один GitHub release entry, общий список assets.
- Claude и Codex packages остаются самостоятельными компонентами в репозитории и build workflows.
- README, changelog, release notes и smoke tests обновлены под unified release model.

## Проверка перед публикацией

- Пройти `docs/SMOKE_TESTS.md` на App 0.0.8.
- Проверить живой Codex bridge scan: `Overview` и `Confirmed facts` должны наполняться из tool results.
- Проверить живой `API + AI Chat` scan: auto-interpretation после scan, русский ответ при `locale=ru`, нормальный развернутый follow-up.
- Проверить, что GitHub Actions прикрепляет app installer, Claude ZIP и Codex ZIP к одному release `v0.0.8`.
- Подтвердить build/packaging на Node.js 22 в нормальном release environment.

## Известные ограничения

- Codex client по-прежнему управляет MCP approval UX; приложение может подсказать, но не может выдать tool permissions автоматически.
- Качество native AI chat зависит от выбранного provider и модели.
- Единый release asset list не отменяет архитектурную самостоятельность app, MCP server и instruction packages.
