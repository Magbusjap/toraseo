:sparkles: feat(app): v0.0.5 — top toolbar + FAQ + update polish

Top toolbar across the entire window with five menu items
(About, Check Updates, Documentation, FAQ, Settings) plus a
GitHub icon. Visible in both onboarding and normal modes so the
user can read About or open settings before dependencies are
satisfied. Documentation and FAQ link straight to GitHub since
no project website exists yet.

New docs/FAQ.md covers install, dependencies, scanning behavior,
updates, privacy, and contributing — six sections pulling
together all the tribal knowledge that previously lived only in
the README and CHANGELOG. The toolbar's «FAQ» button points
here.

Two updater fixes from v0.0.4 dogfooding: stripHtml() helper
cleans the release notes preview (was showing raw <h2>/<p>/<br>
tags from electron-updater's HTML-rendered feed), and
quitAndInstall switches to silent mode so applying an in-app
update no longer flashes the NSIS installer dialog over our own
notification card. First-time installs from a downloaded .exe
still get the standard NSIS UI — that flow doesn't go through
electron-updater.

Layout refactored from horizontal flex to flex-col with the
toolbar pinned on top; pre-flight error toast moved from top-6
to top-16 so it doesn't collide. Versions bumped 0.0.4 → 0.0.5
in package.json and preload's version constant. CHANGELOG
[App 0.0.5] documents the Added/Fixed/Modified breakdown;
roadmap shifted by one slot (i18n now v0.0.6, Skill handshake
v0.0.7, first skill-v* release v0.0.8).
