:paperclip: feat(app): implement hard-dependency detector and onboarding screen

App now requires three components before scanning is unlocked:
Claude Desktop running, ToraSEO MCP registered in config, ToraSEO Skill
installed on disk. Without all three the main UI is replaced with an
onboarding screen with live checkboxes and a button to launch Claude
Desktop. The transition back to normal UI is automatic — no explicit
user click needed once all three are green.

Implements Q1–Q4 from hard-dependency-pivot.md:
- Q1 (Skill distribution): file-based ZIP in ~/.claude/skills/toraseo/.
  Detected via fs.access on SKILL.md.
- Q2 (Claude not running): "Open Claude Desktop" button via
  cross-platform launcher with known-path discovery and claude://
  URI fallback.
- Q3 (polling interval): 5 seconds.
- Q4 (onboarding design): three calm rows with hint text, single
  primary action, no blame in copy.

Race condition between polling tick and user click closed via
pre-flight checkNow(): renderer invokes a synchronous IPC handler
right before starting a scan. If anything dropped within the last
5 seconds → toast + return to onboarding.

New files:
- electron/detector.ts — process scan via ps-list, JSON config read,
  filesystem check; polling loop on 5s interval; checkNow() handler
- electron/launcher.ts — Windows/macOS/Linux known paths plus
  claude:// URI scheme fallback; spawn detached so Claude survives
  ToraSEO closing
- src/hooks/useDetector.ts — subscription to status pushes,
  exposes checkNow() and openClaude() actions
- src/components/Onboarding/OnboardingView.tsx — main screen with
  three DependencyCheck rows and a single primary launcher button
- src/components/Onboarding/DependencyCheck.tsx — one row, satisfied
  state shows green check, missing state shows orange X plus action
- src/components/Onboarding/index.ts — barrel export

Modified files:
